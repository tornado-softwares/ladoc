## Meta options

You can attach extra options to the opening fence to control how a block is rendered.

### Title

Add `title="..."` to display a filename or label above the block.

````md
```ts title="src/index.ts"
console.log('Hello');
```
````

```ts title="src/index.ts"
console.log('Hello');
```

### Line numbers

Add `lineNumbers` to show line numbers down the left side of the block.

````md
```ts lineNumbers
const a = '123';
console.log(a);
```
````

```ts lineNumbers
const a = '123';
console.log(a);
```

### Combining options

Meta options can be combined freely, for example a title together with line numbers:

````md
```ts twoslash title="Test" lineNumbers
type Player = {
  name: string;
};
```
````

```ts twoslash title="Test" lineNumbers
type Player = {
  name: string;
};
```

## Highlighting lines

Wrap the line numbers you want to highlight in curly braces after the language.

````md
```ts {1,3}
const a = 1;
const b = 2;
console.log(a + b);
```
````

```ts {1,3}
const a = 1;
const b = 2;
console.log(a + b);
```

This highlights lines `1` and `3`.

### Diff-style highlighting

Use `ins={...}` and `del={...}` to mark lines as added or removed, alongside a regular `{...}` highlight if needed.

````md
```ts {2} ins={3} del={1}
const oldValue = 1;
const value = 2;
console.log(value);
```
````

```ts {2} ins={3} del={1}
const oldValue = 1;
const value = 2;
console.log(value);
```

## Inline annotations

Instead of (or in addition to) meta options, you can annotate specific lines directly in the code using `// [!code ...]` comments. This is often easier to maintain since the annotation lives next to the line it affects.

````md
```ts
// [!code focus ]
const a = 1;
// [!code highlight ]
const b = 2;
// [!code ++ ]
const added = true;
// [!code -- ]
const removed = false;
// [!code error ]
throw new Error('Oops');
// [!code warning ]
console.warn('Warning');
```
````

```ts
// [!code focus]
const a = 1;
// [!code highlight]
const b = 2;
// [!code ++]
const added = true;
// [!code --]
const removed = false;
// [!code error]
throw new Error('Oops');
// [!code warning]
console.warn('Warning');
```

Available annotations:

| Annotation             | Effect                                      |
| ---------------------- | ------------------------------------------- |
| `// [!code highlight]` | Highlights the next line                    |
| `// [!code focus]`     | Focuses the next line (dims the rest)       |
| `// [!code ++]`        | Marks the next line as added (diff style)   |
| `// [!code --]`        | Marks the next line as removed (diff style) |
| `// [!code error]`     | Marks the next line as an error             |
| `// [!code warning]`   | Marks the next line as a warning            |

### Annotating a range of lines

Append `:N` to any annotation to apply it to the next `N` lines instead of just one.

````md
```ts
const a = 1;
// [!code highlight:2 ]
const b = 2;
const c = 3;
```
````

```ts
const a = 1;
// [!code highlight:2]
const b = 2;
const c = 3;
```

The same range syntax works for every annotation type:

````md
```ts
// [!code focus:4 ]
const a = 1;
const b = 2;
const c = 3;
const d = 4;
```
````

```ts
// [!code focus:4]
const a = 1;
const b = 2;
const c = 3;
const d = 4;
```

```ts

const b = 2;
```
````

```ts

const a = 1;
const b = 2;
```

````md
```ts

const b = 2;

```ts

## Twoslash for TypeScript

Add `twoslash` to the language meta to enable **TypeScript Twoslash**: hover types, inline completions, and error reporting rendered directly from the TypeScript compiler.

### Hover types with `^?`

Place `// ^?` under an identifier to show its inferred type.

````md
```ts twoslash
const a = '123';
console.log(a);
//     ^?
```
````

```ts twoslash
const a = '123';
console.log(a);
//          ^?
```

### Showing compiler errors with `@errors`

Use the `@errors` directive to display specific TypeScript error codes inline instead of failing the build.

````md
```ts twoslash
// @errors: 2588
const a = '123';
a = 132;
```
````

```ts twoslash
// @errors: 2588
const a = '123';
a = 132;
```

### Autocompletion with `^|`

Use `// ^|` to simulate an editor completion popup at a given position, and `// @noErrors` to suppress errors caused by the incomplete snippet.

`````md
````ts twoslash title="Test" lineNumbers
type Player = {
  /**
   * The player name
   * ```php
   * $a = "60"
   * ```
   *
   * @default 'user'
   */
  name: string;
};
// ---cut---
// @noErrors
console.g;
//       ^|
const player: Player = { name: 'Hello World' };
//    ^?
````
`````

````ts twoslash title="Test" lineNumbers
type Player = {
  /**
   * The player name
   * ```php
   * $a = "60"
   * ```
   *
   * @default 'user'
   */
  name: string;
};
// ---cut---
// @noErrors
console.g;
//       ^|

const player: Player = { name: 'Hello World' };
//    ^?
````

> `---cut---` hides everything above it from the rendered output, which is handy for setting up types or imports without cluttering the example.

## Inline code

Wrap text in single backticks for inline code, like `` `This is inlined` ``.

**Markdown:**

```md
And look `This is inlined`
```

**Result:**

And look `This is inlined`

## Supported languages

Since highlighting is powered by Shiki, you can use the language identifier for any language Shiki supports as said in my first words, for example `ts`, `tsx`, `py`, `bash`, `json`, `yaml`, and many more. Just swap the identifier after the opening triple backticks.

```py
a = 0
if (a < 0):
    print("Byebye World")
else:
    print("Hello World")
```
